JavaWeb程序设计-T1（Web应用程序）

一、web概述

1、web概念

web的全称是world wide web，称为全球广域网或万维网。

  • 是一种基于超文本和HTTP的、全球性的、动态交互的、跨平台的分布式图形信息系统。
  • 是建立在Internet上的一种网络服务，为浏览者在Internet上查找和浏览信息提供了图形化的、易于访问的直观界面。

web项目：

在浏览器中利用网址去访问的项目就是web项目

2、程序开发体系结构的分类

2.1 C/S结构

C/S（Client/Server）是基于客户端/服务器的体系结构。

特点：

优点：操作简单上手，有些软件不需要依赖网络；安全性较高。

缺点：需要安装才可以使用，更新维护比较麻烦。

案例：

QQ客户端、微信客户端（手机和电脑上安装的所有软件都是基于C/S结构的）

2.2 B/S结构

B/S（Browser/Server）是基于浏览器/服务器的体系结构。

特点：

优点：无需安装，有浏览器就行；可直接放在万维网上，访问便捷；维护方便

缺点：需要记录地址；需要依赖网络

案例：

淘宝网页版、京东网页版（需要利用网址去访问的网址都是基于B/S结构）

3、web应用程序的分类

3.1 静态网站

静态网站采用html语言编写，放置于Web服务器上，用户通过浏览器直接请求解析显示。

静态网站主要是使用前端开发语言（html、css、javascript）开发的网站，不涉及到和数据库交换数据。

缺点：内容固定不变，改变显示必须修改html代码

3.2 动态网站

动态网站指在html静态页面中嵌入Java、C#、Php等脚本代码，将编写后的页面放入Web服务器，由服务器编译转换为最终html返回给客户端。

动态网站就是运用前端开发语言（html、css、javascript）和后端开发语言（Java、C#、php）共同开发的网站，前端开发语言主要用于页面设计、后端开发语言主要用后端逻辑判断，需要涉及到数据库的数据交换。

三、Tomcat服务器

1、web服务器

不同的动态网页需要不同的Web服务器支持才能正常工作，其职责如下：

（1）Web服务器负责接收用户的请求；

（2）将动态网页进行编译，转换为可执行文件；

（3）将生成的静态html响应给客户端。

2、Tomcat服务器

Tomcat是Apache软件基金会开发的一个web服务器。Tomcat 技术先进、性能稳定，而且免费，已经成为目前学习开发Java Web应用的首选。
