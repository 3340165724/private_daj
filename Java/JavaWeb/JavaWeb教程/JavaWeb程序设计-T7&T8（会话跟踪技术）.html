一、会话跟踪技术概述

1、会话

用户打开浏览器，访问web服务器的资源，会话建立，直到有一方（浏览器或服务器）断开连接，会话结束。

  • 在一次会话中，可以包含多次请求和响应。
  • 同一浏览器和同一服务器同一访问周期里的所有请求和响应都算作一次会话。

2、会话跟踪

一种维护浏览器状态的方法，服务器需要识别多次请求是否来自于同一浏览器，以便在同一次会话的多次请求间共享数据。

会话跟踪技术的常见使用场景：

  • 多页面共享数据
  • 记住账号、密码

3、会话跟踪技术的实现方式

会话跟踪技术的实现方式有两种：

  • 客户端（浏览器）会话跟踪技术：Cookie
  • 服务端会话跟踪技术：Session

它们之间最大的区别：Cookie是存储在浏览器端，而Session是存储在服务器端。

二、Cookie

1、Cookie概念

客户端会话技术，将数据保存到客户端（浏览器）的内存中（默认情况下，当浏览器关闭之后数据就会被销毁），以后每次请求都携带Cookie数据进行访问。

Cookie的作用：存储数据；数据共享

2、Cookie基本使用

2.1 发送Cookie

  • 创建Cookie（服务器）

    Cookie cookie = new Cookie("key","value");

  • 发送Cookie

    xxxxxxxxxx
    response.addCookie(cookie);

2.2 获取Cookie

  • 获取Cookie数组

    xxxxxxxxxx
    Cookie[] cookies = request.getCookies();

  • 获取Cookie对象的值

    xxxxxxxxxx
    cookie.getName();
    cookie.getValue();

3、Cookie实现原理

对于Cookie的实现原理是基于HTTP协议的，其中涉及到HTTP协议中的两个请求头信息：

  • 响应头:set-cookie
  • 请求头: cookie

动态效果示意图下载：https://note.youdao.com/s/8CrFlzM0

  • 当浏览器发送请求给ServletA时，Tomcat服务器会创建cookie对象，并将该对象发送到浏览器，发送时Tomcat会在响应头中添加一行数据Set-Cookie:name=zhangsan

  • 浏览器获取到响应结果后，从响应头中就可以获取到Set-Cookie对应值name=zhangsan，并将数据存储在浏览器的内存中

  • 当浏览器再次发送请求给ServletB时，浏览器会自动在请求头中添加Cookie:name=zhangsan发送给Tomcat服务器

  • request对象会把请求头中cookie对应的值封装成一个个Cookie对象，最终形成一个数组；ServletB通过request对象获取到Cookie[]后，就可以从中获取自己需要的数据

4、Cookie存活时间

Cookie是存储在浏览器内存中的，当浏览器关闭之后，Cookie就会被销毁。

如果需要将Cookie数据长时间存储，可以使用setMaxAge()进行设置：

  • setMaxAge(正数)：将Cookie写入浏览器所在电脑的硬盘，持久化存储，到时间自动删除（单位：秒）
  • setMaxAge(负数)：默认值，Cookie在当前浏览器内存中，当浏览器关闭，则Cookie被销毁
  • setMaxAge(0)：删除对应Cookie

    package com.ynnz.servlet;
    

    import java.io.IOException;
    import java.io.PrintWriter;
    

    import javax.servlet.ServletException;
    import javax.servlet.annotation.WebServlet;
    import javax.servlet.http.Cookie;
    import javax.servlet.http.HttpServlet;
    import javax.servlet.http.HttpServletRequest;
    import javax.servlet.http.HttpServletResponse;
    

    @WebServlet("/login")
    public class Login extends HttpServlet {
        @Override
        protected void service(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
            // 设置相应数据的字符集编码和内容类型
            response.setCharacterEncoding("utf-8");
            response.setContentType("text/html;charset=utf-8");
            
            // 获取登录页面提交的请求数据
            request.setCharacterEncoding("utf-8");
            String acc = request.getParameter("acc");
            String pwd = request.getParameter("pwd");
            // 判断是否登录成功
            if(acc.equals("admin") && pwd.equals("123456")) {
                // 判断复选框是否被选中
                if("on".equals(request.getParameter("check"))) {
                    System.out.println("111");
                    //1. 创建Cookie对象
                    Cookie c_acc = new Cookie("acc",acc);
                    Cookie c_pwd = new Cookie("pwd",pwd);
                    // 设置Cookie的存活时间
                    c_acc.setMaxAge(60 * 60 * 24 * 7);
                    c_pwd.setMaxAge(60 * 60 * 24 * 7);
                    //2. 发送
                    response.addCookie(c_acc);
                    response.addCookie(c_pwd);
                }
                // 进入主界面
                request.getRequestDispatcher("main.jsp").forward(request, response);
            }
            else {
                PrintWriter out = response.getWriter();
                out.print("<script>alert('账号或密码错误')</script>");
                response.sendRedirect("login.jsp");
            }
        }
    }

三、Session

1、Session概念

服务端会话跟踪技术：将数据保存到服务端。

  • 存储在客户端（浏览器）的数据容易被窃取和截获，存在很多不安全的因素，存储在服务端的数据相比于客户端来说就更安全。

2、Session的基本使用

  • 创建Session对象

    xxxxxxxxxx
    HttpSession session = request.getSession();

    • Session对象提供的方法：

      • 存储数据到 session 域中：void setAttribute(String name, Object o)
      • 根据 key，获取值：Object getAttribute(String name)
      • 根据 key，删除该键值对：void removeAttribute(String name)

3、Session实现原理

Session要想实现一次会话多次请求之间的数据共享，就必须要保证多次请求获取Session的对象是同一个。ServletA中存储的数据在ServletB中能够获取到，说明Session对象是同一个。

如何保证ServletA和ServletB中的Session对象是同一个？Session是基于Cookie实现的

  • 动态效果示意图下载：https://note.youdao.com/s/QJj44ST

  • 当浏览器访问ServletA时，Tomcat服务器第一次获取session对象会有一个唯一的标识，假如是id:10

  • Tomcat服务器发现业务处理中使用了session对象，就会把session的唯一标识id:10当做一个cookie，添加Set-Cookie:JSESSIONID=10到响应头中，并响应给浏览器

  • 浏览器接收到响应结果后，会把响应头中的coookie数据存储到浏览器的内存中

  • 当浏览器访问ServletB时，会把cookie中的数据按照cookie: JSESSIONID=10的格式添加到请求头中并发送给服务器Tomcat

  • ServletB获取到请求后，从请求头中就读取cookie中的JSESSIONID值为10，然后就会到服务器内存中寻找id:10的session对象，如果找到了，就直接返回该对象，如果没有则新创建一个session对象

  • 关闭打开浏览器后，因为浏览器的cookie已被销毁，所以就没有JSESSIONID的数据，服务端获取到的session就是一个全新的session对象【所以说：Session是基于Cookie实现的】

4、Session销毁

4.1 默认情况下，无操作，30分钟自动销毁

对于这个失效时间，是可以通过配置进行修改的：

  • 局部配置：在项目的web.xml中配置

  • 全局配置：在Tomcat的web.xml配置文件中修改

4.2 使用invalidate()强制销毁

可以调用Session对象中的invalidate()方法直接销毁Session

  • 该销毁方法一般会在用户退出的时候，需要将session销毁掉。
xxxxxxxxxx
package com.ynnz.servlet;


import java.io.IOException;


import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import javax.servlet.http.HttpSession;


@WebServlet("/a")
public class A extends HttpServlet {
    @Override
    protected void service(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        // 创建Session对象
        HttpSession session = request.getSession();
        
        // 销毁Session对象
        session.invalidate();
        
        // 存储数据
        session.setAttribute("username", "zs");
    }
}

四、Cookie 和 Session的区别

1、存储位置不同

  • Cookie存储在：浏览器端（客户端）
  • Session存储在：服务器端

2、失效时机不同

  • Cookie失效的情况：浏览器或服务器关闭、达到失效时间后会自动销毁（setMaxAge()）
  • Session失效的情况：默认30分钟失效、使用invalidate()立刻销毁session、服务器关闭、浏览器关闭会销毁session编号，而不是直接销毁数据

3、安全性不同

  • session的安全性比cookie的安全性高

4、使用场景不同

  • Cookie使用场景：记住密码（持久化存储：setMaxAge()）
  • Session使用场景：多页面数据共享、退出系统（立刻销毁：invalidate()）
